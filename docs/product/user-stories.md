# User Stories

## Functional Criteria

The app must:

- Work offline
- Optimize battery life
- Be simple to use
- Rely primarily on GPS
- Collect quality data for verification

## Unique Challenges

- Bright sunlight and difficult screen visibility
- Poor connectivity and interrupted uploads
- Field usage in remote/off-grid conditions
- Low-literacy contexts
- Multiple users sharing the same device

## Story Set

1. A user plants a tree and quickly captures one image per tree repeatedly.
2. A user plants many trees and later captures updates at scale, with duplicate risk.
3. A user tends existing trees and captures periodic growth updates.

## Target Users

### Primary

- Organization coordinators who manage many planters and need verifiable reporting.
- Individual planters and village coordinators operating in shared-device workflows.

### Secondary

- Connected smartphone users in developed regions with different usage patterns.
